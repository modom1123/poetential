# WP Frame Search ‚Äî Internal Tool Faceted internal search tool for Warby Parker frames, filterable by lens width, bridge, temple length, frame shape, and style book. ## Files ``` index.html ‚Üê entire app, no dependencies, no build step frames.json ‚Üê frame data (edit this to add/update frames) README.md ``` --- ## Deploy in 3 steps (GitHub Pages) ### 1. Create a new repo Go to github.com ‚Üí New repository ‚Üí name it something like `wp-frame-search` ‚Üí Create. ### 2. Push these files ```bash git init git add . git commit -m "initial" git branch -M main git remote add origin https://github.com/YOUR_USERNAME/wp-frame-search.git git push -u origin main ``` ### 3. Enable GitHub Pages Repo ‚Üí Settings ‚Üí Pages ‚Üí Source: **Deploy from branch** ‚Üí Branch: `main` / `/ (root)` ‚Üí Save. Your app will be live at: `https://YOUR_USERNAME.github.io/wp-frame-search/` --- ## Adding frames ### Option A ‚Äî Edit frames.json (simplest) Open `frames.json`, add an entry following this shape: ```json { "name": "Percey", "sku": "OPT-1042", "lens": 51, "bridge": 18, "temple": 145, "shape": "Rectangle", "colors": ["Jet Black", "Whiskey Tortoise"], "books": ["Rectangle Book", "Heritage Book"], "available": true, "units": 4 } ``` Valid `shape` values: `Rectangle`, `Round`, `Oval`, `Cat-eye`, `Square` Commit and push ‚Äî GitHub Pages deploys automatically within ~60 seconds. ### Option B ‚Äî Use the in-app form Click **+ Add frame** in the top bar. Frames added this way are session-only unless you also connect Firestore (see below). --- ## Connect Firestore (optional ‚Äî enables live data + persistent adds) 1. Go to [console.firebase.google.com](https://console.firebase.google.com) ‚Üí New project 2. Add a **web app** ‚Üí copy the `firebaseConfig` object 3. In Firebase console ‚Üí Firestore Database ‚Üí Create database ‚Üí Start in test mode 4. In `index.html`, find the `FIREBASE SETUP` comment block and uncomment it 5. Paste your config values where indicated 6. Push ‚Äî the app now reads from Firestore and the "+ Add frame" button writes to it Firestore rules for a team-internal tool (paste into Rules tab): ``` rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /frames/{doc} { allow read: if true; allow write: if request.auth != null; } } } ``` To seed Firestore with your existing frames, you can import `frames.json` via the Firebase console or run a one-time script. --- ## Customizing | What | Where | |---|---| | Add a new style book | Add the book name to a frame's `books` array in `frames.json` ‚Äî the sidebar facet builds itself from the data | | Add a new frame shape | Add a path to the `SHAPES` object in `index.html` and add the option to the shape chip list | | Change location badge | Find `loc-badge` in `index.html` ‚Äî it auto-updates to show the data source | | Adjust mm slider ranges | Find `RANGES` in the JS ‚Äî change `lo`/`hi` values |
